The Time Service

 

Welcome to the time service. This document explains how the time services interfaces with the rest of the world. Basically the Time Service is a Custom YAWL Service. You can read about how to create such services here.

 

The Time service will check out a work item from the engine. This work item must contain the data element <time> </time> expressed as a string type. This string can be in several different formats depending on its use.

 

  • Relative time in Milliseconds <number of milliseconds>
  • Relative time in seconds, minutes, etc.. <X> year <Y> mth <Z> day <P> h <Q> m <R> s
  • Absolute date <date format> (e.g. 2/7/2004 13:30:56)
